# リファクタリング記録（2026-06-23）

## 目的

以下3観点で、既存実装の改善を実施した。

- セキュリティ強化（認証まわり）
- 可読性向上（型安全性・責務明確化）
- パフォーマンス/安定性向上（DB接続と取得件数制御）

## 変更内容と理由

### 1) ログイン試行のレート制限を追加

- 変更箇所
  - `backend/app/middleware/login_rate_limit.py`（新規）
  - `backend/app/main.py`
- 内容
  - `POST /auth/jwt/login` に対して、IP単位の試行回数制限を追加
  - 制限超過時に `429 TOO_MANY_REQUESTS` を返却
- 理由
  - ブルートフォース攻撃のリスクを下げるため

### 2) パスワードポリシーを追加

- 変更箇所
  - `backend/app/auth/manager.py`
- 内容
  - 最小文字数（`PASSWORD_MIN_LENGTH`）
  - 英字+数字の両方を必須化
  - （更新時）メールローカル部を含むパスワードを拒否
- 理由
  - 弱いパスワードの登録を抑止するため

### 3) JWT秘密鍵の検証を追加

- 変更箇所
  - `backend/app/config.py`
  - `.env.example`
- 内容
  - `JWT_SECRET` の最小長を検証（64文字以上）
  - プレースホルダ文字列の利用を拒否
- 理由
  - 設定ミスによる脆弱なトークン署名を防止するため

### 4) Request ID ミドルウェアを ASGI 実装へ変更

- 変更箇所
  - `backend/app/middleware/request_id.py`
  - `backend/app/main.py`（ミドルウェア順序調整）
- 内容
  - `BaseHTTPMiddleware` から純ASGI実装へ移行
  - すべてのHTTPレスポンス（429含む）に `X-Request-ID` を付与
- 理由
  - ストリーミング/SSE対応時の互換性とオーバーヘッドを改善するため

### 5) セッションAPIのページネーション追加

- 変更箇所
  - `backend/app/routers/sessions.py`
  - `backend/app/repositories/sessions.py`
  - `backend/app/config.py`
  - `.env.example`
- 内容
  - `GET /api/sessions` に `limit` / `offset` を追加
  - `GET /api/sessions/{id}` のメッセージ取得にも `limit` / `offset` を追加
  - 上限/初期値を設定値化（`*_DEFAULT_LIMIT`, `*_MAX_LIMIT`）
- 理由
  - データ増加時のレスポンス肥大化とDB負荷を抑えるため

### 6) DBプール初期化の排他制御を追加

- 変更箇所
  - `backend/app/database.py`
- 内容
  - `asyncio.Lock` により `init_pool` / `close_pool` の同時実行競合を回避
- 理由
  - 並行アクセス時の二重初期化や状態不整合を防ぐため

### 7) セッションリポジトリの型安全化

- 変更箇所
  - `backend/app/repositories/sessions.py`
  - `backend/app/routers/sessions.py`
  - `backend/app/schemas/sessions.py`
- 内容
  - `dict[str, Any]` を廃止し、`@dataclass` の `SessionRecord` / `MessageRecord` を導入
  - ルーター側も明示型変換へ変更
  - `SessionDetailRead.messages` を `Field(default_factory=list)` に修正
- 理由
  - キー文字列ミスや暗黙的型依存を減らし、保守性を高めるため

### 8) HTTPエラー詳細の可読性を改善

- 変更箇所
  - `backend/app/handlers/errors.py`
- 内容
  - `HTTPException.detail` が dict の場合、`reason` を優先して表示
- 理由
  - パスワードバリデーション失敗などのエラー文言を読みやすくするため

## 環境変数の追加/変更

- 追加
  - `LOGIN_RATE_LIMIT_WINDOW_SECONDS`
  - `LOGIN_RATE_LIMIT_MAX_ATTEMPTS`
  - `PASSWORD_MIN_LENGTH`
  - `SESSION_LIST_DEFAULT_LIMIT`
  - `SESSION_LIST_MAX_LIMIT`
  - `MESSAGE_LIST_DEFAULT_LIMIT`
  - `MESSAGE_LIST_MAX_LIMIT`
- 注意
  - `JWT_SECRET` は64文字以上のランダム値が必須

- 認証
  - `register` / `login` / `users/me` が正常動作
  - 弱いパスワードで `400` になることを確認
- レート制限
  - ログイン試行過多で `429` になることを確認
  - `request_id` と `X-Request-ID` が返ることを確認
- セッションAPI
  - 作成・一覧・詳細が動作
  - `limit` / `offset` クエリで取得件数制御できることを確認
